# Security Policy

## Supported Versions

Security fixes are expected to land on the latest `main` branch state. Older snapshots and forks may not receive patches.

## Reporting a Vulnerability

- Do not post exploit details in a public issue.
- Prefer GitHub's private vulnerability reporting for this repository if it is enabled.
- If private reporting is unavailable, open a GitHub issue that requests a private contact channel without including sensitive details.

Please include:

- affected commit, branch, or release
- reproduction steps
- impact assessment
- any suggested mitigation or workaround

Reports will be triaged as quickly as possible. Public disclosure should wait until a fix or mitigation is available.
